Complete KyOS Guide

Everything you need to know about KyOS Dragon Arch

Introduction to KyOS

KyOS (Dragon Arch) is a modern, security-focused Arch Linux distribution designed for developers, security professionals, and power users. Built around the powerful Qtile tiling window manager, KyOS provides a keyboard-driven, highly customizable desktop environment that maximizes productivity and system control.

What makes KyOS special? KyOS combines the flexibility of Arch Linux with carefully curated tools, sensible defaults, and a beautiful desktop environment that works out of the box.

Key Features

  • Qtile Window Manager: Python-based tiling window manager with powerful customization
  • Security Tools: Pre-installed penetration testing and security analysis tools
  • Developer Ready: Full development environment with modern tools and workflows
  • Rolling Release: Latest packages and features from Arch Linux repositories
  • Minimalist Design: Clean, distraction-free interface focused on productivity

Getting Started

System Requirements

Minimum Requirements

  • 64-bit processor (x86_64)
  • 2GB RAM (4GB recommended)
  • 20GB free disk space
  • USB port or DVD drive
  • Internet connection

Recommended Specifications

  • Modern multi-core processor
  • 8GB+ RAM for development
  • 50GB+ SSD storage
  • Graphics card with 1GB VRAM
  • UEFI firmware support

Download and Verification

  1. Download the latest KyOS ISO from our GitHub releases page
  2. Verify the ISO checksum to ensure integrity
  3. Create a bootable USB drive using tools like Rufus, Balena Etcher, or dd

Verifying ISO Checksum (Linux)

# Download the checksum file
wget https://github.com/rubberpirate/kyos/releases/latest/download/kyos.sha256

# Verify the ISO
sha256sum -c kyos.sha256

Installation Process

Important: Always backup your important data before installing any operating system.

Boot from Live USB

  1. Insert the KyOS USB drive and boot from it
  2. Select "Boot KyOS Dragon Arch" from the boot menu
  3. Wait for the live environment to load
  4. You'll be greeted with the KyOS desktop

Running the Installer

KyOS includes a smart installer that guides you through the installation process:

Start Installation

# Launch the graphical installer
kyos-install

# Or use the automated installer for advanced users
sudo kyosinstall-auto

Installation Steps

  1. Language & Keyboard: Select your preferred language and keyboard layout
  2. Disk Partitioning: Choose automatic or manual partitioning
  3. User Account: Create your user account and set passwords
  4. System Configuration: Configure timezone, hostname, and services
  5. Package Selection: Choose additional software packages
  6. Installation: Watch real-time progress as the system installs
Pro Tip: The installer shows real-time package installation progress, so you can see exactly what's happening during installation.

Understanding Qtile

Qtile is the heart of the KyOS desktop experience. As a tiling window manager, it automatically organizes your windows for maximum screen utilization and productivity.

Core Concepts

  • Workspaces: Virtual desktops for organizing different tasks
  • Layouts: Different ways windows can be arranged (tiled, floating, maximized)
  • Groups: Named workspaces (1-9) for specific purposes
  • Bars: Status bars showing system information and workspace status

Essential Keybindings

Window Management

  • Super + j/k - Move focus between windows
  • Super + h/l - Resize windows
  • Super + Shift + j/k - Move windows
  • Super + f - Toggle fullscreen
  • Super + Shift + q - Close window

Applications

  • Super + Return - Terminal
  • Super + d - Application launcher
  • Super + w - Web browser
  • Super + e - File manager
  • Super + c - Code editor

Customization Guide

Qtile Configuration

Qtile's configuration is written in Python, making it incredibly powerful and flexible:

Configuration File Location

# Main configuration file
~/.config/qtile/config.py

# Backup the original before modifying
cp ~/.config/qtile/config.py ~/.config/qtile/config.py.backup

Common Customizations

Changing Themes

KyOS comes with several built-in themes. Switch between them easily:

# In your config.py, find the colors section
colors = {

}

Adding Custom Keybindings

# Add to the keys list in config.py

Package Management

Pacman Basics

KyOS uses Arch Linux's powerful pacman package manager:

System Updates

# Update package database and system
sudo pacman -Syu

# Update package database only
sudo pacman -Sy

# Force refresh of package database
sudo pacman -Syy

Installing Packages

# Install a package
sudo pacman -S package_name

# Install multiple packages
sudo pacman -S package1 package2 package3

# Install from local file
sudo pacman -U package.pkg.tar.xz

Removing Packages

# Remove package
sudo pacman -R package_name

# Remove package and dependencies
sudo pacman -Rs package_name

# Remove package, deps, and config files
sudo pacman -Rns package_name

Searching and Information

# Search for packages
pacman -Ss search_term

# Get package information
pacman -Si package_name

# List installed packages
pacman -Q

AUR (Arch User Repository)

Access thousands of additional packages through the AUR using yay:

# Install AUR package
yay -S aur_package_name

# Update AUR packages
yay -Syu

# Search AUR
yay -Ss search_term
AUR Safety: Always review PKGBUILDs before installing AUR packages. The AUR contains user-submitted content that isn't officially verified.

Security Tools Suite

KyOS includes a comprehensive collection of security and penetration testing tools:

Network Security

  • Nmap: Network discovery and security auditing
  • Wireshark: Network protocol analyzer
  • Netcat: Network utility for reading/writing network connections
  • Masscan: High-speed port scanner

Web Application Testing

  • Burp Suite: Web application security testing platform
  • OWASP ZAP: Web application security scanner
  • SQLmap: Automatic SQL injection tool
  • Dirb/Gobuster: Directory and file brute-forcing tools

Forensics and Analysis

  • Autopsy: Digital forensics platform
  • Volatility: Memory forensics framework
  • Binwalk: Firmware analysis tool
  • John the Ripper: Password cracking tool
Ethical Use: These tools are for educational purposes, authorized testing, and improving security. Always ensure you have permission before testing any systems.

System Maintenance

Regular Maintenance Tasks

System Updates

Keep your system updated regularly:

# Full system update
sudo pacman -Syu

# Update AUR packages
yay -Syu

Cleaning Package Cache

# Remove old package versions
sudo pacman -Sc

Managing Services

# List running services
systemctl list-units --type=service --state=running

# Enable service at boot
sudo systemctl enable service_name

# Start service now
sudo systemctl start service_name

# Check service status
systemctl status service_name

# View system logs
journalctl

# View logs for specific service
journalctl -u service_name

# View recent logs
journalctl -f

# Clear old logs
sudo journalctl --vacuum-time=2weeks

Troubleshooting Common Issues

Boot Issues

System Won't Boot

  1. Boot from KyOS live USB
  2. Mount your root partition: sudo mount /dev/sdXY /mnt
  3. Chroot into system: sudo arch-chroot /mnt
  4. Reinstall bootloader: grub-install /dev/sdX && grub-mkconfig -o /boot/grub/grub.cfg

Graphics Issues

# Install appropriate graphics drivers
# For NVIDIA
sudo pacman -S nvidia nvidia-utils

# For AMD
sudo pacman -S xf86-video-amdgpu

# For Intel
sudo pacman -S xf86-video-intel

Network Issues

# Check network interface status
ip link show

# Restart NetworkManager
sudo systemctl restart NetworkManager

# Connect to WiFi manually
nmcli device wifi connect "SSID" password "password"

Audio Issues

# Unmute audio
amixer sset Master unmute
amixer sset Master 50%

# Restart audio service
sudo systemctl restart alsa-state

Advanced Topics

Custom Kernel Compilation

For advanced users who need custom kernel features:

# Install kernel build dependencies
sudo pacman -S base-devel

# Download kernel source
git clone https://github.com/archlinux/linux.git
cd linux

# Configure and build
make menuconfig
make -j$(nproc)
sudo make modules_install
sudo make install

System Hardening

  • Configure firewall with ufw or iptables
  • Set up AppArmor or SELinux
  • Configure secure boot
  • Implement disk encryption
  • Harden SSH configuration

Backup Strategies

# System backup with rsync
sudo rsync -aAXv / /backup/location --exclude={"/dev/*","/proc/*","/sys/*","/tmp/*","/run/*","/mnt/*","/media/*","/lost+found"}

# Snapshot with timeshift
sudo timeshift --create --comments "Pre-update snapshot"
