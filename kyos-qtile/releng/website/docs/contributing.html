Contributing to KyOS

Help make KyOS better for everyone

Welcome Contributors!

KyOS is a community-driven project, and we welcome contributions from developers, designers, documentation writers, and users of all skill levels. Every contribution helps make KyOS better for the entire community.

New to Open Source? Don't worry! We have beginner-friendly issues and are happy to help you get started with your first contribution.

Getting Started

1. Set Up Development Environment

# Clone the repository
git clone https://github.com/rubberpirate/kyos.git
cd kyos

# Set up development dependencies
sudo pacman -S git base-devel archiso

# Install AUR helper (if not already installed)
sudo pacman -S --needed base-devel git
git clone https://aur.archlinux.org/yay.git
cd yay && makepkg -si

2. Explore the Codebase

  • releng/ - ISO building configuration
  • releng/airootfs/ - Root filesystem customizations
  • releng/airootfs/usr/local/bin/ - Custom scripts and installer
  • website/ - Project website
  • docs/ - Documentation source

3. Build a Test ISO

# Navigate to releng directory
cd releng

# Build the ISO
sudo mkarchiso -v .

# The ISO will be created in the out/ directory

Contribution Guidelines

Code Style

  • Follow existing code style and conventions
  • Use meaningful variable and function names
  • Comment complex logic and functions
  • Keep functions focused and concise
  • Use proper indentation (4 spaces for Python, 2 for shell scripts)

Commit Messages

# Good commit message format
type(scope): description

# Examples:
feat(installer): add dual boot detection
fix(qtile): resolve bar positioning issue
docs(contributing): update setup instructions
style(website): improve mobile responsiveness

Pull Request Process

  1. Fork the repository to your GitHub account
  2. Create a feature branch from main
  3. Make your changes with clear commit messages
  4. Test your changes thoroughly
  5. Update documentation if needed
  6. Submit a pull request with detailed description

Reporting Bugs

Before Reporting

  • Search existing issues to avoid duplicates
  • Try to reproduce the bug consistently
  • Test with a fresh KyOS installation if possible
  • Gather relevant system information

Bug Report Template

**Bug Description**
A clear description of what the bug is.

**Steps to Reproduce**
1. Go to '...'
2. Click on '....'
3. Scroll down to '....'
4. See error

**Expected Behavior**
What you expected to happen.

**Actual Behavior**
What actually happened.

**System Information**
- KyOS Version: [e.g. 2024.01]
- Hardware: [CPU, RAM, GPU]
- Installation type: [Full disk, dual boot, VM]

**Additional Context**
Any other context about the problem.
